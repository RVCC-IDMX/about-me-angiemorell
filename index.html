CSS ANIMATION

CSS Animation CSS animation is a tool that allows us to manipulate HTML elements. This could be text, images, you name it. You can almost think of this as PowerPoint animations. Transform, hide/show, change the opacity, really anything you can do with CSS you can transition your element from one way to another. Animations can be subtle or in your face but it always has the same intention for the user. Keep the user and engaged focused! Static sites are a thing of the past. Whether you realize it or not, sites you visit every day provide an enjoyable experience with the help of animations.

There are two basic building blocks when it comes to animation:

  1. You have KEYFRAMES which are used to define each and every time an action should be applied to the element being animated. An animation can have multiple stages allowing for a seamless transition from start to finish.
  2. and ANIMATION PROPERTIES which is what is actually happening during these keyframes. Again, these can range from a wide array of CSS rules allowing you to create whatever you can image; with a little work that is.

Keyframes:

As discussed, their role is to define each and every step and style for your animations. There are a few elements a keyframe is made up of.

  1. The name or identifier for the element you are applying these animations to. Generally, you want to be straightforward with what the animation is doing so naming the element and animation appropriately is a must. You wouldn't want to be fooled by a fade animation that's actually bouncing some text into view.
  2. Next you have how long the duration of the animation is. Your animation can be as long or as short as you want it to be, so we can think of the start of the animation as 0% and the end as 100% of signaling. This is an important rule, as within those boundaries you can perform the necessary animation effects or transitions you desire. Timing can be an effective tool. Providing an animation too short can make the viewer wonder if it's finished, while keeping the animation going longer than necessary can make the animation seem out of place or something the user rather skip through.

Animation Properties:

As stated above animation properties can provide a lot of functionality, here is a view:

For example:

  • animation name - The name of the animation, remember be straightforward
  • animation-duration - How long you intend the animation to run
  • animation timing function - This allows you to specify how fast or slow you want the animation to go. However, you don't need just one speed. Using this you can change the speed at different intervals creating a curve.
  • animation-delay - This can be used to specify a delay before the animation will start

These are just to name a few!
